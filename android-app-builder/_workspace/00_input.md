# 입력 요구사항 정리

## 원문 요약
- Android 앱 제작
- 서버 없이 사용자 스마트폰 내부에서 동작(온디바이스 중심)
- 비 오는 날 우산을 어느 방향으로 기울여야 하는지 안내
- 한국 서비스 기준의 날씨 API 활용
- 최적 모델(기술/알고리즘) 선정 후 단계별 설계

## 해석된 제품 목표
- 실시간 강수 + 바람 데이터를 이용해 빗방울 유입 방향을 계산한다.
- 사용자 헤딩(기기 방향)을 기준으로 상대 방향을 직관적으로 제시한다.
- 개인정보 최소화: 위치/센서 데이터는 기기 내부 처리, 외부 전송은 API 호출에 필요한 좌표만 사용.

## 범위
- 포함: Android 네이티브(Kotlin + Compose), 위치/방향 센서, 날씨 API 연동, 우산 방향/기울기 계산 로직
- 제외: 지도 기반 기준물 안내, 자체 서버 구축, 실시간 사용자 추적 백엔드, 계정/로그인

## 성공 기준
- 현재 위치 기준으로 5초 이내 첫 안내 표시
- 비가 없을 때 "우산 기울임 불필요" 상태 명확히 표시
- 비가 있을 때 우산 기울임 방향(각도)과 권장 기울임 각도(도) 제공
