# 01 UX 설계

## 제품명(작업명)
- RainLean (가칭)

## 핵심 사용자 시나리오
1. 사용자가 앱 실행 후 위치/방향 권한을 허용한다.
2. 앱이 현재 강수 여부와 바람 방향/풍속을 조회한다.
3. 앱이 우산을 어느 방향으로 얼마나 기울일지 즉시 제시한다.
4. 사용자는 3D처럼 기울어지는 우산 시각 요소를 보며 각도를 조정한다.

## 화면 구성
| 화면명 | 목적 | 주요 컴포넌트 |
|---|---|---|
| 온보딩/권한 | 권한 획득 및 기능 설명 | 권한 카드, 시작 버튼 |
| 실시간 안내(메인) | 즉시 방향 안내 | 우산 시각화, 각도 카드, 상태 배지 |
| 설정 | API/단위/알림 설정 | 스위치, 드롭다운, **배너 알림 토글** |
| 오류/복구 | 네트워크/센서 오류 대응 | 재시도 버튼, 대체 안내 |
| 배너 알림(시스템) | 백그라운드 우산 방향 전달 | 시스템 알림 채널(무음), 우산 이모지 아이콘 |

## 메인 화면 정보 구조
- 상단: 강수 상태(`비 없음`, `약한 비`, `강한 비`)
- 중앙: 우산 기울임 방향(예: `좌전방 30도`) + 권장 기울임 각도(예: `18도`)
- 하단: 보조 안내(시계방향 라벨, 센서 안정성 문구)
- 보조: 신뢰도 배지(`높음/보통/낮음`)와 마지막 갱신 시각

## 상태 정의
- `NoRain`: 강수 0 또는 임계치 미만, 기울임 불필요
- `RainReady`: 강수 존재 + 풍향 데이터 정상
- `LowConfidence`: 강수는 있으나 풍향/센서 품질 낮음
- `Error`: 위치/네트워크/센서 오류
- `BannerActive`: 앱 내 토글 ON + 알림 권한 허용 → 배너 알림 표시 중
- `BannerPermissionDenied`: `POST_NOTIFICATIONS` 거부됨 → Snackbar로 설정 링크 안내

## 배너 알림 콘텐츠 명세

앱에서 배너 알림 스위치를 ON으로 설정하면 즉시 활성화.

| 요소 | 내용 |
|---|---|
| 알림 채널 | `umbrella_banner` (IMPORTANCE_LOW, 무음·무진동) |
| 소형 아이콘 | `ic_umbrella_stat` (단색 벡터, 상태바용) |
| 대형 아이콘 | 우산 이모지(☂)를 `relativeDirectionDeg` 각도로 Canvas 회전한 Bitmap |
| 제목 | `강수 {mm} mm/h · 풍속 {mps} m/s` |
| 본문(`NoRain`) | `비 예보 없음 · 우산 기울임 불필요` |
| 본문(`RainReady`) | `우산을 {방향} {각도}° 기울이세요` |
| 본문(`LowConfidence`) | `신뢰도 낮음 · {방향} {각도}° 참고용` |
| 갱신 주기 | 날씨 데이터: 15분 (WorkManager) / 표시 방향: 5초 |
| 탭 동작 | `MainActivity` 열기 (`PendingIntent`) |
| 지속 여부 | `setOngoing(true)` — 사용자가 직접 밀어서 닫기 불가 |
| 마지막 갱신 표시 | 제목에 `· 갱신 HH:mm` 추가(LowConfidence/오래된 데이터 시) |

## 접근성
- 색상 외 텍스트/아이콘 동시 제공
- 진동 패턴으로 방향 변경 알림(옵션)
- 큰 글씨 모드 대응(Material 3 dynamic type)
