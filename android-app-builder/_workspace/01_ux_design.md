# 01 UX 설계

## 제품명(작업명)
- RainLean (가칭)

## 핵심 사용자 시나리오
1. 사용자가 앱 실행 후 위치/방향 권한을 허용한다.
2. 앱이 현재 강수 여부와 바람 방향/풍속을 조회한다.
3. 앱이 우산을 어느 방향으로 얼마나 기울일지 즉시 제시한다.
4. 사용자는 3D처럼 기울어지는 우산 시각 요소를 보며 각도를 조정한다.

## 화면 구성
| 화면명 | 목적 | 주요 컴포넌트 |
|---|---|---|
| 온보딩/권한 | 권한 획득 및 기능 설명 | 권한 카드, 시작 버튼 |
| 실시간 안내(메인) | 즉시 방향 안내 | 우산 시각화, 각도 카드, 상태 배지 |
| 설정 | API/단위/알림 설정 | 스위치, 드롭다운 |
| 오류/복구 | 네트워크/센서 오류 대응 | 재시도 버튼, 대체 안내 |

## 메인 화면 정보 구조
- 상단: 강수 상태(`비 없음`, `약한 비`, `강한 비`)
- 중앙: 우산 기울임 방향(예: `좌전방 30도`) + 권장 기울임 각도(예: `18도`)
- 하단: 보조 안내(시계방향 라벨, 센서 안정성 문구)
- 보조: 신뢰도 배지(`높음/보통/낮음`)와 마지막 갱신 시각

## 상태 정의
- `NoRain`: 강수 0 또는 임계치 미만, 기울임 불필요
- `RainReady`: 강수 존재 + 풍향 데이터 정상
- `LowConfidence`: 강수는 있으나 풍향/센서 품질 낮음
- `Error`: 위치/네트워크/센서 오류

## 접근성
- 색상 외 텍스트/아이콘 동시 제공
- 진동 패턴으로 방향 변경 알림(옵션)
- 큰 글씨 모드 대응(Material 3 dynamic type)
