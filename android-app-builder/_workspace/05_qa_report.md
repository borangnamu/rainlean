# 05 QA 보고서 (설계 단계)

## 검증 범위
- 요구사항 반영 여부
- 무서버(온디바이스) 제약 충족 여부
- API 의존성/리스크
- 알고리즘 설명 가능성

## 점검 결과
| 항목 | 결과 | 메모 |
|---|---|---|
| 서버 미사용 구조 | 통과 | 외부 API 직접 호출 구조 |
| 비/바람 기반 우산 방향 | 통과 | 계산 유스케이스 정의 완료 |
| 지도 의존성 제거 | 통과 | 지도 SDK/랜드마크 추천 흐름 제거 |
| 한국 서비스 적합성 | 통과 | KMA 중심 구성 |
| 예외 처리 | 통과 | 권한/네트워크/센서 오류 전략 정의 |

## 잔여 리스크
- API 키가 앱에 포함되므로 역공학 위험 존재
- 도심 협곡/돌풍 환경에서 체감 방향 오차 가능
- 일부 기기에서 센서 품질 편차 존재

## 권고 테스트 케이스
1. 비 없음(강수 0) 상태에서 안내 비활성화 확인
2. 강풍+강수 상태에서 기울임 각도 상한(30도) 동작 확인
3. 기기 흔들림 상황에서 회전 안정화 동작 확인
4. 센서 오차가 큰 기기에서 신뢰도 배지 하향 확인
5. 네트워크 끊김 시 캐시/오류 메시지 처리 확인
