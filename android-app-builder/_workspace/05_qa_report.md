# 05 QA 보고서 (설계 단계)

## 검증 범위
- 요구사항 반영 여부
- 무서버(온디바이스) 제약 충족 여부
- API 의존성/리스크
- 알고리즘 설명 가능성

## 점검 결과
| 항목 | 결과 | 메모 |
|---|---|---|
| 서버 미사용 구조 | 통과 | 외부 API 직접 호출 구조 |
| 비/바람 기반 우산 방향 | 통과 | 계산 유스케이스 정의 완료 |
| 지도 의존성 제거 | 통과 | 지도 SDK/랜드마크 추천 흐름 제거 |
| 한국 서비스 적합성 | 통과 | KMA 중심 구성 |
| 예외 처리 | 통과 | 권한/네트워크/센서 오류 전략 정의 |

## 잔여 리스크
- API 키가 앱에 포함되므로 역공학 위험 존재
- 도심 협곡/돌풍 환경에서 체감 방향 오차 가능
- 일부 기기에서 센서 품질 편차 존재
- **Xiaomi(MIUI) / Samsung(One UI) 등 OEM 기기에서 백그라운드 앱 킬러로 Foreground Service 강제 종료 가능**
- **Android 14+ 에서 백그라운드 상태의 ForegroundService 시작 제한 (BootReceiver 경로 주의)**
- **5초 갱신이 배터리 제한(Standby Bucket) 상태에서 스로틀링될 수 있음**

## 권고 테스트 케이스

### 기존 기능
1. 비 없음(강수 0) 상태에서 안내 비활성화 확인
2. 강풍+강수 상태에서 기울임 각도 상한(30도) 동작 확인
3. 기기 흔들림 상황에서 회전 안정화 동작 확인
4. 센서 오차가 큰 기기에서 신뢰도 배지 하향 확인
5. 네트워크 끊김 시 캐시/오류 메시지 처리 확인

### 배너 알림 (신규)
6. **권한 허용 경로**: Android 13+ 기기에서 배너 토글 ON → `POST_NOTIFICATIONS` 요청 다이얼로그 표시 → 허용 → 배너 즉시 표시 확인
7. **권한 거부 경로**: 요청 거부 → 토글 OFF 복귀 + Snackbar "설정 열기" 표시 확인
8. **권한 영구 거부**: "다시 묻지 않음" 후 토글 ON → 앱 알림 설정(`ACTION_APP_NOTIFICATION_SETTINGS`) 링크 표시 확인
9. **배너 표시 내용 검증**: 비 상태(`RainReady`)일 때 우산 이모지가 풍향 기준으로 회전, 제목/본문 형식 확인
10. **5초 갱신**: 기기를 회전시키며 5초 이내 배너 아이콘 방향 변경 확인
11. **15분 날씨 갱신**: Logcat `WeatherRefreshWorker` 로그로 갱신 확인 (실기기 16분 대기)
12. **배터리 절약 모드**: 도즈(Doze) 상태에서 15분 Worker 최대 수 분 지연 허용 여부 확인
13. **재부팅 후 복구**: 배너 ON 상태에서 기기 재부팅 → `BootReceiver`로 WorkManager 재등록, 배너 재표시 확인
14. **OEM 백그라운드 킬러**: Xiaomi/Samsung 에서 배터리 최적화 예외 설정 없이 15분 이상 백그라운드 후 Worker 동작 확인
15. **비 없음 상태 배너**: `NoRain` 상태일 때 배너 본문 "비 예보 없음 · 우산 기울임 불필요" + 회전 없는 아이콘 확인
16. **네트워크 단절 + 배너**: 비행기 모드 전환 후 배너가 마지막 캐시 데이터로 유지되는지 확인
17. **토글 OFF**: 배너 ON 상태에서 토글 OFF → 알림 즉시 사라지고 WorkManager 취소 확인
