# App Store Optimization — Google Play ASO 가이드

store-manager 에이전트가 Google Play 배포 시 활용하는 메타데이터 최적화, 키워드 전략, 정책 대응 레퍼런스.

## 대상 에이전트

`store-manager` — 이 스킬의 ASO 전략과 가이드라인을 배포 문서에 직접 적용한다.

## 메타데이터 최적화 매트릭스

| 필드 | 길이/형식 | ASO 영향 | 작성 포인트 |
|------|-----------|-----------|-------------|
| 앱 이름 | 30자 | 최상 | 핵심 키워드 1개 포함 |
| 짧은 설명 | 80자 | 최상 | 가치 제안과 주요 기능을 한 문장에 압축 |
| 전체 설명 | 4,000자 | 높음 | 상단 5줄에 핵심 가치 배치 |
| 스크린샷 | 최대 8장 | 높음 | 첫 3장이 전환에 가장 중요 |
| 프로모션 영상 | 선택 | 중간 | 핵심 가치 30초 이내 요약 |
| 카테고리 | 1개 | 중간 | 검색 의도와 가장 맞는 분류 선택 |

## 키워드 전략

1. 핵심 기능에서 시드 키워드를 뽑는다
2. 동의어와 검색 의도를 기준으로 확장 키워드를 만든다
3. 설명 상단과 스크린샷 카피에 핵심 키워드를 자연스럽게 배치한다
4. 과도한 반복 대신 문장 자연스러움을 유지한다

## 스크린샷 가이드

| 순서 | 내용 | 목적 |
|------|------|------|
| 1장 | 핵심 가치 제안 | 왜 필요한 앱인지 설명 |
| 2장 | 주요 기능 1 | 가장 자주 쓰는 흐름 강조 |
| 3장 | 주요 기능 2 | 차별화 포인트 설명 |
| 4장 | 보조 기능 | 반복 사용 이유 강화 |
| 5장 | 신뢰 요소 | 안정성, 개인화, 리뷰 포인트 |

## Google Play 주요 리젝 사유

| 사유 | 대응 |
|------|------|
| 크래시/ANR | 테스트 근거와 안정성 점검 추가 |
| 권한 과다 요청 | 최소 권한 원칙과 사용 시점 요청 |
| 데이터 안전 섹션 누락 | Data safety 양식과 개인정보처리방침 일치 |
| 타깃 API 미준수 | 최신 타깃 SDK 기준 명시 |
| 지적재산권 문제 | 타사 로고, 이미지, 상표 사용 점검 |

## 설명 작성 공식

```text
[핵심 가치 한 문장]

[핵심 기능 3~5개 불릿]

[차별화 포인트]

[신뢰 요소 또는 사용 시나리오]

[지금 다운로드해야 하는 이유]
```
